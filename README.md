# AI 工作交接套装

> 一键生成交接文档，让下一个 AI 无缝接手

AI 对话上下文满了？任务做到一半要换窗口？这个工具帮你生成交接文档，下一个 AI 直接接手，无需重复说明。

---

## 快速开始

### 安装

```bash
# 方法一：skill.sh 一键安装（推荐）
git clone https://github.com/noendings/ai-handover.git
bash ai-handover/skill.sh

# 方法二：手动安装
git clone https://github.com/noendings/ai-handover.git
cp -r ai-handover/skills/ai-handover ~/.claude/skills/

# 方法三：npm
npm install @anthropic-ai/claude-code-skill-ai-handover
```

### 三个命令

| 命令 | 作用 | 场景 |
|-----|------|-----|
| `/gzjj` 工作交接 | 生成交接文档 | 对话快满了，需要开新窗口 |
| `/xrtk` 新人填坑 | 接手工作 | 新对话，继续之前的任务 |
| `/ai-handover` | 显示功能菜单 | 查看所有选项 |

> `/gzjj` 和 `/xrtk` 是拼音首字母缩写（工-作-交-接、新-人-填-坑），`skill.sh` 安装时会自动创建这两个命令别名。

### 一分钟上手

**场景：对话上下文快满了**

```
你: /gzjj

AI: 请选择存储位置：
    A. 当前文件夹
    B. 项目根目录（推荐）

你: B

AI: 生成交接文档...
    time_3_基于index.html的OCR简谱识别交接文档-20260424.md

    ✅ 已验证文件存在
    ❓ 需要你确认：Vercel 环境变量是否已配置？
```

**场景：新对话继续工作**

```
你: /xrtk

AI: 请选择要接手的工作：
    1. time_3_基于index.html的OCR简谱识别交接文档-20260424.md
    2. time_2_基于editor.html的简谱编辑器交接文档-20260423.md

你: 1

AI: 我对当前情况的理解：
    项目：OCR简谱识别
    进度：70%
    卡点：Vercel 环境变量未配置

    接下来怎么干？
    A. 直接开干
    B. 先看代码
```

## 核心功能

### 1. 工作交接 (gzjj)

分析当前对话上下文，自动生成包含：
- 任务背景与需求演变
- 已完成工作与代码修改记录
- 当前进度可视化
- 卡点与阻塞问题
- 用户画像与沟通建议
- 下一步行动计划

**状态验证**：自动检查文件是否存在、代码能否运行，区分"用户声称完成"和"实际验证通过"。

### 2. 新人填坑 (xrtk)

读取交接文档，快速理解：
- 项目全貌与核心诉求
- 历史决策与代码位置
- 当前卡点与待办事项
- 立即可执行的行动

---

## 为什么需要这个

**问题**：AI 是个概率放大器，输出 token 越多错越多。上下文一换，之前的理解就丢了。

**解法**：不让上下文丢。每完成一个小任务，让 AI 总结交接文档，下一个 AI 直接继承。

**核心思路**：
```
对话快满 → /gzjj 生成交接文档 → 开新对话 → /xrtk 接手工作 → 继续
```

## 文档规范

### 命名规则

```
交接文档：time_{x}_基于{入口文件}的{功能简述}交接文档-{时间戳}.md
填坑文档：time_{x}_新人填坑_理解规划_{项目名}_{时间戳}.md
```

编号相同 = 配套关系，支持 Obsidian 双链语法。

### 状态标记

| 标记 | 含义 |
|-----|------|
| ✅ | 已验证通过 |
| ✅(用户确认) | 用户声称完成，未自动验证 |
| 🔄 | 进行中 |
| ⏳ | 待开始 |
| ❓ | 状态不明 |
| ❌ | 已失败 |
